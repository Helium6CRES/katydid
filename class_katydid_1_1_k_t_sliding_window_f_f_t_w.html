Detailed Description

Creates a 2-D (frequency vs. time) power spectrum from an bundle.

Author
N. S. Oblath

Slides a window along the length of the (time-domain) data, performing Fourier Transforms as the window moves. The distance the window slides between transforms is given by the overlap or overlap fraction.

  • The overlap is the number of bins or time units of overlap between the window from one transform and the next.
  • The overlap fraction gives the overlap between the window from one transform and the next as a fraction of the window length. Any type of window function (inheriting from KTWindowFunction) can be used.

Available configuration values:

  • "transform-flag": string – flag that determines how much planning is done prior to any transforms
  • "use-wisdom": bool – whether or not to use FFTW wisdom to improve FFT performance
  • "wisdom-filename": string – filename for loading/saving FFTW wisdom
  • "overlap-time": float – sets the overlap in time units
  • "overlap-size": int – sets the overlap in number of bins
  • "overlap-frac": float – sets the overlap in fraction of the window length
  • "window-function-type": string – sets the type of window function to be used
  • "window-function": subtree – parent node for the window function configuration
  • "input-data-name": string – name used to find data when processing bundles
