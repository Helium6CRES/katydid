1 /*
2  * KTHDF5TypeWriterTransform.hh
3  @details
4 
5 Saves FFTs and Power Spectra to HDF5 files.
6 
7 All FFTs from a given Egg or Mat file are saved inside a single HDF5 dataset (instead of one dataset per slice).
8 
9 The dataset will have 4 dimensions:
10 1 - Number of channels (sometimes called fNComponents in other modules)
11 2 - Number of slices
12 3 - Number of parts (real-only: 1; real and imaginary: 2; polar amplitude and angle: 2)
13 4 - Number of samples in slice
14 
15 The slot determines what FFT or Power Spectrum will be written to file.
16 itâ€™s one unique slot for each possible output:
17 fs-fftw -> complex FFT
18 fs-polar -> polar FFT
19 ps -> Power Spectrum
20 psd -> PSD
21 etc...
22 
23 Inputs:
24 output-file: full path with name for output file. Preferred extension is h5.
25 use-compression: true or false (default: false). Uses the built-in hdf5 compression.
26 
27 Example:
28  ...
29  { "type": "hdf5-writer", "name": "h5w" },
30  ...
31  {
32  "signal": "fft:fft",
33  "slot": "h5w:fs-fftw"
34  }
35  ...
36  "h5w":
37  {
38  "output-file": "/temp/fft_20150523T1206_TopCoil1p0A_CF1050MHz_000000001_to_000000014.h5",
39  "use-compression": true
40  }
41 
42 
43  @author: J.N. Kofron,
44  @date: 9/12/2014
45  @edited: 6/17/2015, L. de Viveiros
46  */
